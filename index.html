1. Compute & Serverless

AWS Lambda - kien thuc cot loi

Lambda chay code khong can quan ly server. Moi invocation chay trong mot execution environment (sandbox MicroVM). Vong doi gom 2 pha:

  • Init phase (allocation time): tai code, load layers, chay code o global scope, khoi tao SDK client, DB connection. Pha nay tinh phi nhung KHONG tinh trong duration cua handler khi co Provisioned Concurrency.
  • Invoke phase: chay handler. Cold start = lan dau khoi tao moi truong. Warm start = tai su dung moi truong san co.

Cold start - cach giam

  • Provisioned Concurrency: giu sang execution environment, chay init truoc, dam bao latency on dinh. Chi gan duoc cho version cu the hoac alias, KHONG gan cho $LATEST. Phai publish version moi gan duoc.
  • Reserved Concurrency: chi gioi han so concurrent execution (quota), khong giam cold start.
  • Dat code init ra global scope (ngoai handler) de tai su dung connection, SDK client giua cac invocation tren cung container.
  • Tang memory dong nghia tang CPU/network -> giam tong thoi gian thuc thi (vd: xu ly anh, thumbnail).

Lambda VPC access

  • Mac dinh Lambda khong nam trong VPC -> khong truy cap duoc resource private (RDS, EC2 private subnet).
  • De truy cap: attach Lambda vao VPC + private subnets + security group. Lambda tao ENI trong subnet, dung Hyperplane ENI quan ly chia se.
  • Khong dung NAT Gateway de Lambda noi voi resource trong VPC (NAT chi cho outbound Internet).
  • Khong dung VPC Endpoint de Lambda noi EC2 (Endpoint danh cho AWS service nhu S3, DynamoDB).

Lambda + RDS / Aurora

  • Surge traffic + nhieu Lambda mo connection -> dung RDS Proxy de pool connection, tranh exhaust max_connections.
  • Best practice: mo DB connection o global scope (ngoai handler) de reuse.

Lambda timeout vs SQS visibility timeout

  • Khi Lambda trigger bo SQS, neu xu ly lau hon Visibility Timeout cua queue -> message tro lai queue, bi xu ly trung.
  • Quy tac: SQS Visibility Timeout >= Lambda timeout + buffer, va Visibility Timeout phai du dai cho ca call ngoai (vd 60s goi API ML).
  • Tang Lambda memory/timeout KHONG sua duoc trung lap; phai chinh visibility timeout.

Lambda Destinations (async)

  • Khi Lambda invoke async (vd tu SNS), co the cau hinh Success destination va Failure destination = SQS / SNS / EventBridge / Lambda.
  • Khac voi DLQ: DLQ chi nhan event THAT BAI sau khi retry het. Destinations co them success + ghi du metadata.
  • De log moi invocation -> dung 2 destination (success + failure) cung tro ve 1 SQS.

Lambda alias + weighted routing (canary/staging)

  • Alias tro toi 1 hoac 2 version voi % traffic (weighted). Cho phep canary deploy: vi du 90% prod version, 10% new version.
  • SDK/API Gateway goi alias staging -> Lambda tu split traffic.
  • Khong gan weight cho version truc tiep, chi alias moi co weight.

Lambda /tmp ephemeral storage

  • Moi execution environment co thu muc /tmp, mac dinh 512 MB, co the tang toi 10 GB.
  • Tot cho workload doc lap nhieu lan tu mot file lon (vd download video tu S3, xu ly nhieu pass). Doc local nhanh hon nhieu so voi S3 GetObject lap lai.
  • Lambda khong gan EBS, khong gan ENA - chi co /tmp hoac mount EFS.

Lambda Insights vs X-Ray

  • CloudWatch Lambda Insights: theo doi performance noi tai (CPU, memory, init duration, network). Bat bang cach: them Lambda Insights Extension Layer + gan policy CloudWatchLambdaInsightsExecutionRolePolicy.
  • AWS X-Ray: tracing distributed, do latency giua Lambda va downstream service (S3, DynamoDB...). Tren service map, click vao edge/line giua 2 node de xem latency bao gom network.
  • Logs Insights chi query log; @duration la tong thoi gian Lambda, khong tach rieng downstream.

AWS CodeDeploy - Deployment Strategies

  • Canary: shift 1 phan nho truoc (vd 10% trong 5 phut), neu OK thi roll 100% cung mot luc -> phu hop "thu nhom nho roi deploy dong loat".
  • Linear: shift theo % deu dan (vd 10% moi 10 phut).
  • All-at-once: deploy het cung luc.
  • In-place vs Blue/Green: in-place update truc tiep instance dang chay; blue/green tao moi truong moi va switch.

Amazon ECS Fargate

  • Fargate khong quan ly EC2 host. Container chay tren ha tang AWS managed.
  • Shared persistent storage giua nhieu task -> dung Amazon EFS qua efsVolumeConfiguration va mountPoints trong task definition.
  • EBS gan vao EC2 instance, khong gan tu nhien cho Fargate task (co support EBS volume cho task tu 2024 nhung khong share duoc).
  • DockerVolumeConfiguration chi ho tro EC2 launch type, KHONG ho tro Fargate.
  • ECS Task IAM Role (bat ECS_ENABLE_TASK_IAM_ROLE=true): cap quyen rieng cho moi task, dap ung least privilege; tach biet voi EC2 instance profile role (cho ECS agent).

2. Storage - S3, EBS, EFS, ElastiCache

Amazon S3 - kien thuc cot loi

S3 Versioning + Lifecycle

  • Khi bat Versioning, moi PutObject tao mot version moi. Object cu thanh noncurrent version.
  • De gioi han so version giu lai (vd current + 1 truoc), dung S3 Lifecycle rule: NoncurrentVersionExpiration voi NewerNoncurrentVersions.
  • Bucket policy chi quan ly access, khong quan ly retention.
  • Object Lock = WORM (compliance), khong dung de gioi han version.

S3 Encryption - phan biet ro

LoaiAi quan ly keyDac diem
SSE-S3S3 tu quan ly (AES-256)Don gian nhat, khong audit duoc
SSE-KMSAWS KMS (CMK)Audit qua CloudTrail, kiem soat quyen kms:Decrypt
SSE-CCustomer cung cap keyS3 khong luu key
DSSE-KMS2 lop ma hoa, KMSWorkload compliance cao
Client-sideClient ma hoa truoc khi uploadDung GenerateDataKey + envelope encryption
Mau cau hoi: "Encrypt BEFORE upload" -> client-side encryption voi kms:GenerateDataKey. Lambda goi GenerateDataKey nhan plaintext data key + CiphertextBlob, ma hoa file bang plaintext key, store CiphertextBlob trong metadata. "Encrypt AT REST" -> SSE-S3 / SSE-KMS.

In-transit encryption (S3)

  • HTTPS bao mat kenh truyen. De ENFORCE HTTPS cho bucket: dung bucket policy voi "aws:SecureTransport": "false" -> Deny.
  • Cai cert tren EC2, dung VPC Endpoint... KHONG enforce HTTPS o phia S3.
{
  "Effect": "Deny",
  "Principal": "*",
  "Action": "s3:*",
  "Resource": ["arn:aws:s3:::bucket","arn:aws:s3:::bucket/*"],
  "Condition": {"Bool": {"aws:SecureTransport": "false"}}
}

S3 Object Lambda

  • Cho phep bien doi data on-the-fly khi client goi GetObject thong qua S3 Object Lambda Access Point.
  • Use case: redact PII, resize anh, convert format ma khong can luu nhieu ban copy hoac refactor consumer.
  • Khac biet: ma hoa (SSE) KHONG che PII (decrypt xong van thay); muon che PII thi can transformation.

S3 storage classes - lien quan latency

  • S3 Standard: truy cap tuc thi, latency thap, gia cao hon.
  • S3 Glacier / Deep Archive: can restore tu vai phut den vai gio -> KHONG dap ung yeu cau "trong 5 phut".
  • S3 Transfer Acceleration: tang toc upload/download qua CloudFront edge, KHONG phai cho content delivery + language logic.

S3 message pattern voi SQS

  • SQS message gioi han 256 KB. Voi file 10 MB: luu file vao S3, gui S3 location (key/URL) vao SQS de consumer xu ly bat dong bo.

Amazon EFS

  • File system NFS managed, hieu nang co the scale, share giua nhieu EC2/ECS task.
  • Ho tro Fargate mount qua task definition (efsVolumeConfiguration).

Amazon EBS

  • Block storage gan vao 1 EC2 instance (gp3 default).
  • Khong share trong cung Region (tru io2 Block Express multi-attach gioi han).
  • Khong dung cho Lambda hoac Fargate share storage.

Amazon ElastiCache

  • In-memory cache, latency microsecond.
  • Memcached: don gian, multi-thread, khong persistence, dung cho session cache.
  • Redis: persistence, pub/sub, replication, cluster mode.
  • Use case dien hinh: tach session state khoi EC2 web server (stateless tier) -> dap ung scale-out + giam tai memory. Lua chon "lowest-latency session store" -> ElastiCache.
  • So sanh: DynamoDB, RDS deu cao latency hon ElastiCache.

Tip: Khi co GSI voi partition key la field can loc, luon dung Query tren GSI de tiet kiem RCU. Scan + FilterExpression van phai doc het roi moi loc -> dat.

DynamoDB Streams

  • Phat su kien khi co thay doi tren item: INSERT, MODIFY, REMOVE.
  • Stream view type: KEYS_ONLY, NEW_IMAGE, OLD_IMAGE, NEW_AND_OLD_IMAGES.
  • Khi can du lieu sau khi update (vd dong bo sang Google Sheet, ElasticSearch) -> chon NEW_IMAGE.
  • Trigger Lambda consume stream de xu ly downstream.

DAX (DynamoDB Accelerator)

  • In-memory cache cho DynamoDB, giam latency doc tu ms xuong microsecond.
  • Chi tang toc doc, khong phat stream/event. Khong thay the duoc Streams.

Amazon RDS / Aurora

  • Aurora MySQL/PostgreSQL: managed, scale storage tu dong, replica nhanh.
  • Khi co nhieu Lambda concurrent connect -> dung RDS Proxy (connection pooling, su dung IAM auth, lay credential tu Secrets Manager).
  • Connection bao mat: dung Secrets Manager rotation; Lambda fetch credential tai runtime (khong dung env var de tu cap nhat).
  • Encryption at-rest cua RDS khong tu giai ma ung dung level (vd CloudFront Field-Level Encryption).

Amazon CloudFront

  • CDN toan cau, cache tai edge, giam latency.
  • Field-Level Encryption (FLE): ma hoa cac field nhay cam tai edge bang RSA public key (tu KMS), origin/ung dung phai decrypt bang private key truoc khi luu DB. Neu khong decrypt -> DB se nhan ciphertext (loi thuong gap).
  • Headers dac biet:
    • CloudFront-Viewer-Country: ma quoc gia cua user.
    • Accept-Language: ngon ngu trinh duyet user.
    Forward 2 header nay len origin de phuc vu content theo dia ly + ngon ngu.
  • Lambda@Edge: chay logic JS/Python tai edge, dung cho FLE custom, A/B testing, header manipulation.

Amazon API Gateway

JWT khong code custom -> HTTP API + JWT authorizer. WebSocket khong ho tro JWT built-in.

Amazon Route 53

  • Alias record tro toi ALB, CloudFront, S3 website, API Gateway.
  • Routing policies: simple, weighted, latency, failover, geolocation, geoproximity, multi-value.
  • Geolocation chi route theo quoc gia, KHONG xu ly language header.

AWS Global Accelerator

  • Toi uu duong di TCP/UDP qua AWS backbone, anycast IP.
  • KHONG cache content (khac CloudFront). Phu hop game/IoT/realtime.

VPC

  • VPC Flow Logs: ghi metadata IP traffic in/out trong VPC. Dung de xac dinh request co toi AWS network hay chua (vd ALB timeout).
  • VPC Endpoint: private connect toi AWS service (S3, DynamoDB, KMS,...) khong qua Internet.
  • PrivateLink: chia se service giua VPC/account.
  • NAT Gateway: cho outbound Internet tu private subnet.

5. Security & Identity

IAM

  • Least privilege: cap dung quyen can thiet, scope theo Resource ARN.
  • DynamoDB read actions: GetItem, BatchGetItem, Query, Scan.
  • DynamoDB write actions: PutItem, UpdateItem, DeleteItem, BatchWriteItem.
  • Trust policy voi aws:SourceArn = ARN chinh xac cua resource (vd state machine cua Step Functions) -> chi resource do moi assume duoc role.
  • Tranh dung "*" hoac StringNotEquals ngu canh sai -> cap qua rong.

AWS KMS

  • Customer managed key: tu quan ly key policy, cho phep cross-account access. AWS managed key khong sua duoc policy.
  • Cross-account secret access: can ket hop IAM policy (Secrets Manager) o app account + KMS key policy (kms:Decrypt, kms:DescribeKey) o key owner account.
  • GenerateDataKey: tra ve plaintext data key + CiphertextBlob -> dung cho envelope/client-side encryption. Plaintext key ma hoa data, CiphertextBlob luu kem object de decrypt sau.
  • KMS keys KHONG replicate cross-account; phai tao key moi va cap policy.

AWS Secrets Manager

  • Luu credential DB, API key, connection string. Ho tro automatic rotation.
  • Lambda fetch tai runtime de luon dung gia tri moi (khong nhung qua env var -> env var dong bang luc init).
  • Lambda Extension cua Secrets Manager: caching o moi truong runtime.

AWS Systems Manager Parameter Store

  • Luu config va secret (SecureString = ma hoa qua KMS).
  • Re hon Secrets Manager, nhung khong co rotation built-in cho DB credential.

AWS AppConfig

  • Quan ly feature flags + config dynamic.
  • AppConfig Agent tren EC2/Lambda Extension poll dinh ky, cache local, app doc qua localhost endpoint -> overhead toi thieu, rollback an toan.

Amazon Cognito

ComponentVai tro
User PoolAuth: sign-up, sign-in, MFA, password policy, recovery, phat hanh JWT
Identity PoolCap temporary AWS credentials (qua STS) de goi truc tiep AWS service
  • Mobile app sign in + access backend -> User Pool + JWT goi API Gateway/Lambda.
  • Social login (Google, Facebook, Apple, Amazon) -> User Pool ho tro federation (web identity federation).
  • Mobile app MFA: bat MFA tren User Pool, Cognito tu gui SMS/TOTP - khong code SNS Publish.
  • Khong dung IAM users cho end-user.

AWS STS

  • AssumeRole: nhan temporary credentials de test quyen role hoac cross-account.
  • Regional endpoints (sts.<region>.amazonaws.com) giam latency va tang resilience so voi global endpoint.
  • AssumeRoleWithSAML: federation enterprise (ADFS, Okta). AssumeRoleWithWebIdentity: OIDC (Cognito, Google,...).

AWS Directory Service

  • AWS Directory Service for Microsoft AD: managed AD trong AWS, ho tro trust relationship voi on-prem AD.
  • Nhan vien dang nhap AWS Console bang AD credentials hien co, IAM role tro toi cac group AD -> RBAC.
  • It overhead hon viec tu xay identity broker hoac SAML federation thu cong.

AWS WAF, AWS Config, Cross-cutting

  • WAF: loc request HTTP/HTTPS (SQLi, geo match...). KHONG ma hoa noi dung.
  • AWS Config + managed rule acm-certificate-expiration-check: phat hien cert ACM (bao gom imported certs) sap het han. Ket hop EventBridge rule + SNS de canh bao.
  • Su kien "ACM Certificate Approaching Expiration" cua EventBridge chi cho ACM-managed certs (khong cho imported certs).

6. Messaging & Integration

Amazon SNS

  • Pub/Sub topic, fan-out tin nhan toi nhieu subscriber (SQS, Lambda, HTTPS, SMS, email).
  • Standard topic: best-effort ordering, at-least-once.
  • FIFO topic: ordering (MessageGroupId) + deduplication. Phu hop nghiep vu can dam bao thu tu va khong trung (vi du SMS xac nhan giao dich).
  • Subscription filter policies: loc message tai subscription level (theo message attribute). Tot nhat cho fan-out co dieu kien (so voi loc trong code consumer).
  • S3 -> SNS -> nhieu SQS voi filter policy = pattern fan-out chuan, khong can sua Lambda code.

Amazon SQS

  • Standard queue: throughput cao, ordering best-effort, at-least-once.
  • FIFO queue: dam bao thu tu va exactly-once, throughput thap hon.
  • Visibility Timeout: thoi gian message bi "an" sau khi consumer doc. Lambda timeout phai <= visibility timeout - buffer.
  • Long polling (WaitTimeSeconds 1-20): giam empty receive, tiet kiem chi phi.
  • Batching: ReceiveMessage co the lay toi 10 message/lan -> giam request, giam chi phi.
  • DLQ: nhan message that bai sau khi het maxReceiveCount; chi log loi, khong log success.
  • SQS message max 256 KB; voi payload lon dung SQS Extended Client + S3 (luu S3, gui reference).

EventBridge / EventBridge Pipes / Scheduler

  • EventBridge Rules: pattern match event tu AWS service hoac custom -> target (Lambda, SNS, SQS, Step Functions...).
  • EventBridge Scheduler: lich hen gio (cron/rate) - chi de invoke, KHONG dieu phoi workflow theo phu thuoc.
  • EventBridge Pipes: pipe source -> filter -> enrichment -> target. Khong gui SMS truc tiep.

AWS Step Functions

  • Orchestration workflow (Amazon States Language). Phu hop khi can chay nhieu Lambda theo thu tu, voi retry/error handling, parallel, choice...
  • So sanh: SNS/SQS khong dam bao thu tu giua nhieu function khac nhau theo chain logic.

Amazon Kinesis Data Firehose

  • Streaming ETL toi S3, Redshift, OpenSearch, HTTP endpoint.
  • Ho tro data transformation via Lambda: bien doi/redact PII truoc khi ghi xuong destination.
  • Khong gui SMS truc tiep, khong target la Step Functions/EC2.

7. CI/CD & IaC

AWS CodePipeline

  • Orchestration pipeline: Source -> Build -> Test -> Deploy.
  • Tich hop CodeCommit/GitHub/S3 (source), CodeBuild (build/test), CodeDeploy, CloudFormation, Step Functions (deploy/orchestration).

AWS CodeBuild

  • Managed build server. Chay script tuy y theo buildspec.yml: build, unit test, integration test, package artifact.

AWS CodeDeploy

  • Deploy app len EC2/On-Prem, Lambda, ECS.
  • Service role can policy AWSCodeDeployRole (EC2/On-Prem) hoac AWSCodeDeployRoleForECS. Loi IAM_ROLE_PERMISSIONS -> dinh kem dung policy.
  • Deployment config: Canary, Linear, AllAtOnce; tra ve traffic shifting cho Lambda alias.

AWS CloudFormation

  • IaC khai bao bang JSON/YAML.
  • DeletionPolicy: mac dinh Delete. Co the dat Retain hoac Snapshot de bao toan resource khi update/delete stack.
  • Doi ten resource (logical ID hoac Name property) trong template = CloudFormation xem do la resource moi -> tao moi + xoa cu (theo DeletionPolicy).
  • Change sets: preview thay doi truoc khi apply, KHONG phai cong cu test code local.
  • StackSets: deploy stack tren nhieu account/region mot lan -> phu hop multi-account.

AWS SAM (Serverless Application Model)

  • Extension cua CloudFormation cho serverless.
  • sam local invoke / sam local start-api: test Lambda + API Gateway tren may local.
  • Tich hop CodePipeline: dung SAM CLI trong cac stage de build, package, deploy.

8. Monitoring & Observability

Amazon CloudWatch

  • Logs: thu log tu Lambda, EC2 (CloudWatch agent), on-premises (CloudWatch agent cai tren may + IAM credentials).
  • Logs Insights: query log voi cu phap rieng. Co the dung @duration cho Lambda log nhung do la tong thoi gian, khong tach downstream.
  • Embedded Metric Format (EMF): log JSON theo dac ta EMF -> CloudWatch tu trich xuat custom metrics vao namespace tuy chon (khong can goi PutMetricData rieng).
  • Lambda Insights: extension layer + IAM policy (CloudWatchLambdaInsightsExecutionRolePolicy) -> theo doi performance Lambda (memory, CPU, init duration).
  • CloudWatch RUM: monitoring frontend (real user monitoring), KHONG dung de tao metric tu Lambda log.

AWS X-Ray

  • Distributed tracing. Bat tren Lambda (active tracing) hoac SDK instrumentation.
  • Service map: cac node (service) va edge (line giua chung). Click edge = latency bao gom network giua 2 service (vd Lambda -> S3). Click node = chi do latency cua service do.
  • Active tracing cho SNS: do latency publish -> deliver, dung de phat hien rate va latency cua FIFO topic.

VPC Flow Logs

  • Ghi IP traffic in/out cua ENI. Dung de kiem tra request co toi VPC chua, traffic bi accept/reject.

AWS Config

  • Theo doi config va compliance cua resource. Managed rules vd acm-certificate-expiration-check, s3-bucket-public-read-prohibited.
  • Ket hop EventBridge rule (Config Rules Compliance Change) + SNS = canh bao tu dong.

AWS CloudTrail

  • Ghi API call tren account (audit). KHONG ghi performance/latency cua service.

10. Cach cac service ket noi voi nhau (cheat-sheet)

  • S3 -> Lambda: S3 Event Notification (s3:ObjectCreated:*) hoac qua SNS/SQS/EventBridge.
  • S3 -> SNS -> nhieu SQS: fan-out voi subscription filter policy.
  • API Gateway -> Lambda: REST/HTTP API tich hop Lambda proxy; HTTP API ho tro JWT authorizer.
  • Lambda -> RDS private: Lambda attach VPC + private subnet + SG; them RDS Proxy de pool.
  • Lambda async -> destination: Success/Failure destination = SQS, SNS, EventBridge, Lambda.
  • SQS -> Lambda: event source mapping; Visibility timeout phai >= Lambda timeout.
  • SNS FIFO -> SMS/SQS/Lambda: ordering + dedup, ho tro fan-out.
  • DynamoDB Streams -> Lambda: NEW_IMAGE/OLD_IMAGE/NEW_AND_OLD/KEYS_ONLY.
  • Kinesis Firehose -> Lambda transform -> S3: redact/transform truoc khi luu.
  • CloudFront -> Origin (ALB/S3/Lambda function URL): forward header (Accept-Language, CloudFront-Viewer-Country) de origin tra content phu hop.
  • Step Functions -> Lambda/ECS/SNS/SQS: orchestration tuan tu/song song.
  • CodePipeline -> CodeBuild -> CloudFormation/StackSets/SAM/CodeDeploy: CI/CD chuan.
  • Config -> EventBridge -> SNS: phat hien compliance change + thong bao.
  • Cognito User Pool -> API Gateway authorizer: bao ve backend API bang JWT.
  • Cognito Identity Pool -> STS -> S3/DynamoDB: cap temporary AWS credentials cho client.
  • KMS GenerateDataKey -> Client encrypt -> S3 PutObject: client-side envelope encryption.
  • Secrets Manager -> Lambda fetch runtime -> RDS: credential luon moi sau rotation.
  • AppConfig -> AppConfig Agent (sidecar) -> EC2/Lambda localhost: feature flag polling.
